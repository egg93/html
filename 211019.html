29. 객체 예고

Object
함수와 객체가 따로 존재하는 것이 아닌 함수 기반 위에 객체가 존재.
객체의 존재 이유에 대해 집중.
객체 또한 정리정돈의 수단 => 함수 또한 많아지고 함수와 연관되어 있는 변수 또한 많아지면 결국 또 복잡도의 한계에 봉착. => 그 해결책으로 함수가 등장.

<script>
function setColor(color){
 var aList = document.querySelectorAll('a');
 var i = 0;
 while (i<aList.length) {
  aList[i].style.color=color;
  i = i+1;
 }
}
function nightDayHandler(self){
 var target = document.querySelector('body');
 if (self.value === 'this.night'){
  target.style.backgroundColor='black';
  target.style.color='white';
  self.value='this.day';
  setColor('powderBlue');
  } else {
  target.style.backgroundColor='white';
  target.style.color='black';
  self.value='this.night';
  setColor('blue');
  }
}
</script>

비교적 단순한 로직일지라도 후에 유지보수를 위해 함수를 통해 로직에 분명한 이름을 붙여주는 것 또한 좋은 방법임.

function setColor(color){
 var aList = document.querySelectorAll('a');
 var i = 0;
 while (i<aList.length) {
  aList[i].style.color=color;
  i = i+1;
 }
}
function setColor(color){
 target.style.color=color;
}

이름이 충돌하는 문제점 발생. 이전 코드가 새로운 코드로 덮어씌워지게 됨.
코드가 복잡해지면 동일한 함수명이 존재하는지 하나하나 체크하는 것은 사실상 불가능. 중요한 프로젝트에서 이전 함수가 덮어씌워지면 큰 문제로 확산 가능.

서로 연관된 함수와 변수들을 그룹핑해서 사용할 수 있는 도구 => 객체
일종의 폴더라고 봐도 무방

document.querySelector('body').style.color=color;
document라는 객체에 속해 있는 querySelector()라는 함수라는 것을 유추할 수 있음.
객체에 속해 있는 함수 => method

30. 객체 쓰기와 읽기

정보의 양이 많아졌을 때 서로 연관된 정보를 정리정돈하기 위해 배웠던 도구
배열 => 순서에 따라서 정보를 정리 정돈한다는 특징
순서 없이 정보를 전달할 수 있는 도구 => Object
그러나 체계없이 무작위로 집어넣는다면 그것은 정리정돈 수납함이 아닌 쓰레기통과 별반 다를 바 없음 => labeling을 통해 구분 => 이름이 있는 정리정돈 상자

문법

<script>
var food = {
 "banana": "banana",
 "mango": "mango"
};
document.write("banana: "+food.banana+"<br>");
document.write("mango: "+food.mango+"<br>");
</script>


배열은 [], 객체는 {} 데이터 추가
food.strawberry = "strawberry";
데이터 추가 - 띄어쓰기 있을 시
food["potato chip"] = "potato chip";

이름에는 공백을 넣을 수 없음. 그럴 때는 []를 이용.

31. 객체와 반복문

객체에 담겨 있는 모든 데이터를 가져와야 할 때가 있음.
추천 검색어: javascript object iterate
for...in

<script>
for(var key in food){
 document.write(key+"<br>");
}
</script>


"banana" => key (가져올 수 있는 정보에 도달할 수 있는 열쇠): "banana"
cf) 배열에서는 순서가 있다 하여 index라는 표현을 씀
객체에 담겨 있는 키값을 하나하나 꺼내서 변수값으로 세팅해서 중괄호 안으로 담겨줌.

<script>
for(var key in food){
document.write(food[key]+"<br>");
}
</script>



키의 값이 아닌 키에 담겨 있는 데이터의 값을 꺼내고자 할 때 => 객체명[key명]

32. 객체프로퍼티와 메소드

뒤로 갈수록 몰라도 괜찮다. 앞으로 갈수록 모르면 큰일나는 거다.

property & method

<script>
 food.showAll = function(){
  for(var key in this){
   document.write(key+": "+this[key]+"<br>");
  }
 }
</script>
(= function food.showAll(){} 와 같은 표현식과 다를바 없다.)

var key in this => 객체의 명이 바뀌어도 유지보수가 용이

객체에 소속된 함수 => method
객체에 소속된 변수 => property
